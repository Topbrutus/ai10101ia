# Politique de sécurité

## Signalement

- ouvrir une issue `décision / arbitrage` si la vulnérabilité peut être discutée publiquement
- sinon, traiter le sujet hors issue publique selon le canal privé retenu par l’équipe

## Règles minimales

- aucun secret dans le dépôt
- aucun secret dans le corps des issues ou PR
- aucune commande destructive sans approbation et sans trace
- aucun changement de politique de sécurité sans justification

## Portée

Cette politique couvre :

- scripts de validation
- workflows GitHub Actions
- gabarits PR et issues
- politique de tâches
- documentation d’exploitation
